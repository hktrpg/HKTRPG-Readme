# 事件功能

![](<../../.gitbook/assets/image (31).png>)

`.event (add delete show) .evt (random/事件名称)`

经由添加的事件，会得到一些状态或增加减少经验值，并可以赚取额外经验值。

### 使用教学

`.event add` 详情看下面说明 - 添加事件 \
`.event delete (事件名称)` - 删除事件 \
`.event show` - 显示你添加的所有事件, 及赚取了的EXP \
`.event show (事件名称)` - 显示你添加的指定事件详情 \
`.event useExp` - 在群组中使用, 将会得到你赚取的EXP\
`.evt random` - 进入随机的事件, 消耗5EN \
`.evt (系列名称)` - 进入指定的系列事件, 消耗10EN \
`.evt (事件名称)` - 进入指定的事件, 消耗15EN

### 规则

EN上限 = 20+LV 每10分钟回复1点EN

#### 得知事件名称的方法

别人告知 或 经随机事件知道名字 设计事件的好处 能够吸收对方消耗的en和经验值 作为自己赚取到的经验值

### 添加事件的格式范例

`.event add` \
`name:Haha chain:开心系列` \
`exp:SAN`\
`0:你今天的运气真好;你是个好人;我爱你` \
`-1:你中招了;你不好运要-SAN了` \
`1:你吃了好味的糖，加SAN`

#### name ->&#x20;

事件标题&#x20;

#### chain->&#x20;

系列名称，别人可以指定该系列来进行抽选&#x20;

#### exp ->&#x20;

(可选)经验值的名称, 例如改成SAN, 会变成「你损失了X点SAN」&#x20;

#### 0:你今天的运气真好;你是个好人;我爱你&#x20;

\-> (事件类型):(事件的描述);(事件的描述2);.....(事件的描述N)&#x20;

#### 事件的描述->

会从描述1,2,N中随机选取其中一个

#### 事件类型 ->&#x20;

&#x20; 0\. 没有事发生

1. 直接增加X点经验
2. 未来X次里会得到 X 倍经验值
3. 赠送群组所有人1点经验
4. 赠送作者已赚取到的经验给玩家
5. 从整个CHANNEL 的X人吸收X点经验 -1. 直接减少X点经验 -2. 停止得到经验(X次) -3. 被事件开发者吸收X点经验 -4. 分发X经验给整个CHANNEL中的X人 -5. 每次发言减少X经验(X次内)

***

#### 限制

A. 一个事件中，正面选项要比负面选项多 \
B. 一个事件中，可以有3+(ROUNDDOWN 设计者LV/10) 项选项 \
C. 一个事件中，不可以全部正面效果 \
D. 一个事件可用的总EN 为(10+LV)，负面事件消耗X点EN
